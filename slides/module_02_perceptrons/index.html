Module 2: Perceptrons and Optimization

From Neurons to Networks

Review: Module 1

Shannon Entropy

Information is measurable. The entropy of a source measures the average surprise per symbol:

$$H(X) = -\sum_{i} p(x_i) \log_2 p(x_i)$$

A good model assigns high probability to likely events — low surprise, few bits.

N-gram Models

Predict the next symbol from the previous \(n - 1\) symbols:

$$P(w_k \mid w_{k-n+1} \ldots w_{k-1})$$

Higher order = more context = better predictions, but the number of possible contexts explodes exponentially.

Review: Cross-Entropy as a Loss

In Module 1, we used cross-entropy to evaluate how well an n-gram model predicts text:

$$H(p, q) = -\sum_{x} p(x) \log q(x)$$

Today, the same formula becomes our training objective. Instead of just measuring model quality, we will minimize cross-entropy to make the model learn.

Key connection: The evaluation metric from Module 1 is the loss function for Module 2. Lower cross-entropy = better predictions = a model that has learned more structure.

The Neuron Model

Weighted sums, activations, and the simplest learning machine

A Loose Biological Analogy

Biological Neuron

  • Dendrites receive signals from other neurons
  • Cell body integrates incoming signals
  • Axon fires an output if the signal is strong enough

Artificial Neuron

  • Inputs \(\mathbf{x}\) are multiplied by weights \(\mathbf{w}\)
  • Weighted sum plus a bias \(b\)
  • Activation function \(\sigma\) produces the output

The analogy is useful for intuition but breaks down fast. Biological neurons use spike timing, complex dendritic computation, and diverse neurotransmitter systems — none of which are captured here.

The Perceptron

A single neuron computes:

$$y = \sigma(\mathbf{w} \cdot \mathbf{x} + b)$$

\(\mathbf{w} \cdot \mathbf{x}\)

Dot product of inputs and weights — how much each input matters

\(+ \; b\)

Bias term — shifts the decision boundary

\(\sigma(\cdot)\)

Activation function — introduces nonlinearity

This is a linear function followed by a nonlinearity. That pattern is the fundamental building block of every neural network.

Activation Functions

Step (original perceptron)

$$\sigma(z) = \begin{cases} 1 & z \geq 0 \\ 0 & z < 0 \end{cases}$$

Binary output. Not differentiable at \(z = 0\).

Sigmoid

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Smooth, output in \((0, 1)\). Interpretable as probability. Vanishing gradients when \(|z|\) is large.

Tanh

$$\sigma(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}$$

Zero-centered, output in \((-1, 1)\). Same vanishing gradient issue.

ReLU

$$\sigma(z) = \max(0, z)$$

Cheap to compute. Avoids vanishing gradients. Dominates modern networks.

Why Nonlinearity Matters

Without an activation function, stacking layers collapses to a single linear transformation:

$$W_2(W_1 \mathbf{x} + b_1) + b_2 = \underbrace{W_2 W_1}_{W'} \mathbf{x} + \underbrace{W_2 b_1 + b_2}_{b'}$$

No matter how many linear layers you stack, the result is always a single linear function. Adding layers without nonlinearity gains nothing.

The activation function between layers is what gives neural networks their power. Each nonlinear transformation lets the network represent increasingly complex relationships.

George Cybenko

George Cybenko

Proved the Universal Approximation Theorem

  • Professor at Dartmouth College
  • Proved (1989) that a feedforward network with one hidden layer and sufficient neurons can approximate any continuous function on a compact set
  • The theorem says such a network exists — not that gradient descent will find it

Universal Approximation Theorem

A feedforward network with a single hidden layer containing a finite number of neurons can approximate any continuous function on a compact subset of \(\mathbb{R}^n\), given sufficient neurons.

The caveat: The theorem is existential. It does not guarantee that gradient descent will find the right weights, that the required number of neurons is tractable, or that the network will generalize to new data.

Analogy: A Fourier series can approximate any periodic function, but computing the right coefficients for an arbitrary function is nontrivial. The universal approximation theorem plays the same role for neural networks.

Module 1 connection: N-gram models are also universal approximators of language given enough order. But they don't scale — the number of parameters grows exponentially with context length.

Networks as Matrices

Why GPUs matter

Matrix-Graph Equivalence

A neural network is a directed graph. Each layer's connections can be represented as a weight matrix.

Forward Pass

$$\mathbf{h} = \sigma(W\mathbf{x} + \mathbf{b})$$
  • \(W\) — weight matrix
  • \(\mathbf{x}\) — input vector
  • \(\mathbf{b}\) — bias vector
  • \(\sigma\) — elementwise nonlinearity

Why GPUs?

Matrix multiplication is embarrassingly parallel: each output element is an independent dot product.

A GPU has thousands of cores optimized for exactly this operation. Neural network training is dominated by matrix multiplications.

Module 1 connection: Jensen Huang and NVIDIA bet that GPU computing would become essential. This is why that bet paid off.

Minsky & Papert

Killed Neural Networks for a Decade

  • Published Perceptrons (1969) — proved single-layer nets cannot compute XOR, parity, or connectedness
  • The result was widely interpreted as showing neural networks were fundamentally limited
  • Funding dried up for over a decade — the first AI winter for connectionism
  • Minsky was co-founder of MIT's AI Lab and a leading proponent of symbolic AI
  • They knew multi-layer nets could solve these problems. The critique was that no one knew how to train them.

The XOR Problem

The limitation that froze a field

Linear Separability

A single perceptron defines a hyperplane in the input space — a line in 2D, a plane in 3D.

AND

0, 00
0, 10
1, 00
1, 11

One line separates them.

OR

0, 00
0, 11
1, 01
1, 11

One line separates them.

XOR

0, 00
0, 11
1, 01
1, 10

No single line works.

XOR is not linearly separable — you cannot draw one line to separate the classes. This is what Minsky and Papert proved formally in 1969.

Solving XOR with a Hidden Layer

Two neurons in a hidden layer can each learn a linear boundary. The output neuron combines them.

Hidden Neuron 1

Learns one boundary (e.g., \(x_1 + x_2 > 0.5\))

Hidden Neuron 2

Learns another boundary (e.g., \(x_1 + x_2 < 1.5\))

Output Neuron

Combines them with AND-like logic: output is 1 only when both hidden neurons agree.

This is the simplest demonstration of why depth matters. One layer cannot do what two layers can.

Folding in High-Dimensional Space

A single perceptron splits the input space with a hyperplane. Each hidden layer applies a nonlinear transformation that folds the space.

Visual intuition: Imagine a sheet of paper with two colors of dots mixed together. A single fold (one hidden layer) can bring same-colored dots together. Multiple folds (a deep network) can handle arbitrarily complex arrangements.

  • Points that were entangled can be separated
  • Points that were distant can be brought together
  • With enough folds (layers), you can separate arbitrarily complex class boundaries

Backpropagation

The chain rule, applied systematically

Loss Functions

Before we can train, we need a way to measure how wrong our predictions are. The loss function quantifies the gap between prediction and truth.

Mean Squared Error (Regression)

$$L = \frac{1}{N}\sum_{i=1}^{N}(y_i - \hat{y}_i)^2$$

Penalizes large errors quadratically. Standard for predicting continuous values.

Binary Cross-Entropy (Classification)

$$L = -\frac{1}{N}\sum_{i=1}^{N}\left[y_i \log(\hat{y}_i) + (1 - y_i)\log(1 - \hat{y}_i)\right]$$

Measures surprise under the model's predictions. High loss when confident and wrong.

Module 1 connection: This is Shannon's entropy applied as a training objective. Cross-entropy appeared in Module 1 as an evaluation metric for n-gram models. Here, the same formula drives learning.

The Backpropagation Algorithm

Forward Pass

Compute the output of each layer sequentially. Store intermediate values.

Input \(\rightarrow\) Hidden \(\rightarrow\) Output \(\rightarrow\) Loss

Backward Pass

Starting from the loss, compute \(\frac{\partial L}{\partial w}\) for each weight by propagating gradients backward through the computation graph.

Loss \(\rightarrow\) Output \(\rightarrow\) Hidden \(\rightarrow\) Weights

For sigmoid + BCE, the gradients simplify elegantly:

$$\frac{\partial L}{\partial w_j} = (\hat{y} - y) \cdot x_j \qquad \frac{\partial L}{\partial b} = \hat{y} - y$$

The gradient is just the prediction error times the input. This is why sigmoid + BCE is such a natural pairing.

Without Backprop

The naive alternative: perturb each weight by a small \(\epsilon\), recompute the loss, and estimate the gradient numerically:

$$\frac{\partial L}{\partial w} \approx \frac{L(w + \epsilon) - L(w)}{\epsilon}$$

This requires one forward pass per weight — \(O(n)\) passes for \(n\) weights. You cannot perturb multiple weights simultaneously because you would have no way to attribute the loss change to individual parameters.

For modern networks with billions of parameters, this is completely infeasible. Backpropagation computes all gradients in one backward pass — roughly the same cost as one forward pass.

Computation Graphs

Frameworks like PyTorch build a dynamic computation graph during the forward pass.

  • Each operation records its inputs and the function used
  • The backward pass walks this graph in reverse, applying the chain rule at each node
  • Users define only the forward pass — the framework derives the backward pass automatically

This is called automatic differentiation (autograd). It is the reason you can prototype complex architectures in a few lines of code without manually deriving gradients.

Gradient Descent

Walking downhill on the loss surface

The Update Rule

Compute the gradient of the loss with respect to all weights, then take a step in the negative gradient direction:

$$w \leftarrow w - \eta \frac{\partial L}{\partial w}$$

\(\eta\) too large

Overshoots the minimum and may diverge entirely

\(\eta\) too small

Converges very slowly, may get stuck in local minima

\(\eta\) just right

Steady convergence toward a good minimum

Stochastic Gradient Descent

Instead of computing the gradient over the entire dataset (batch gradient descent), SGD computes it over a random subset (mini-batch).

Small batch (e.g., 32)

  • Noisier gradient estimates
  • Faster per step
  • Noise helps escape local minima

Large batch (e.g., 256)

  • Smoother gradient estimates
  • More memory required
  • May converge to sharper minima

Typical batch sizes: 32, 64, 128, 256. The trade-off between noise and stability is a practical tuning decision.

Adam Optimizer

Adam (Adaptive Moment Estimation) maintains per-parameter learning rates based on the history of gradients.

  • Tracks the first moment (mean) and second moment (variance) of each parameter's gradient
  • Parameters with consistently large gradients get smaller learning rates
  • Parameters with small or noisy gradients get larger learning rates

Adam is the default optimizer in practice for most deep learning tasks. When in doubt, start with Adam.

Loss Landscape

The loss function defines a surface in weight space. For a network with \(n\) weights, this is a surface in \((n + 1)\)-dimensional space.

Sharp Minima

Small weight perturbations cause large loss changes. Tend to generalize poorly.

Flat Minima

Robust to small perturbations. Tend to generalize better to new data.

Saddle Points

Gradient is zero but not a minimum. In high dimensions, far more common than local minima.

The learning rate affects which kind of minimum the optimizer finds. SGD noise helps escape sharp minima and saddle points.

Li et al. 2018, "Visualizing the Loss Landscape of Neural Nets"

Overfitting and Generalization

A network with enough parameters can memorize any training set — achieving zero training loss but failing on new data.

Overfitting

  • Training loss goes to zero
  • Test loss stays high or increases
  • The model has memorized the training data, not learned the pattern

Generalization

  • Low loss on both training and test data
  • The model has learned the underlying structure
  • Performs well on data it has never seen

Regularization techniques (named here, covered in later modules): dropout (randomly zero out neurons during training) and weight decay (penalize large weights with \(\lambda \|\mathbf{w}\|^2\)).

Extra Credit: MLP Backpropagation

Implement mlp_gradients() — backpropagation through both layers of the MLP.

# Forward pass (save intermediates)
z1 = W1 @ x + b1
hidden = sigmoid(z1)
z2 = W2 @ hidden + b2
output = sigmoid(z2)

# Output layer gradients
error_out = output - y_true
dW2 = error_out[:, None] * hidden[None, :]
db2 = error_out

# Hidden layer gradients (backpropagate through sigmoid)
error_hidden = (W2.T @ error_out) * hidden * (1 - hidden)
dW1 = error_hidden[:, None] * x[None, :]
db1 = error_hidden

This is the chain rule applied twice: once through the output layer, once through the hidden layer. The term hidden * (1 - hidden) is the derivative of sigmoid.

Quiz

Test your understanding

Q1: Why Nonlinearity?

A student proposes building a "deep" network with 100 linear layers (no activation functions) to model a complex relationship.

What will this network compute, and why is the student's approach fundamentally flawed?

Answer: The composition of any number of linear transformations is still a single linear transformation: \(W_{100} \cdots W_2 W_1 \mathbf{x} = W' \mathbf{x}\). The 100-layer network has exactly the same representational power as a single layer. Without nonlinear activation functions between layers, depth adds nothing.

Q2: The XOR Barrier

In the exercise, the single neuron achieved 50% accuracy on XOR-like data, with the loss stuck at 0.693.

Why exactly 50%, and what is special about the number 0.693?

Answer: 50% is random chance for a binary classification problem — the neuron cannot do better than guessing. The loss 0.693 is \(\ln(2) \approx 0.6931\), which is the cross-entropy of a fair coin (maximum uncertainty for two classes). The neuron converges to predicting ~0.5 for everything because no linear boundary can improve on random guessing for XOR-structured data.

Q3: Gradient Descent Trade-offs

You are training a neural network and notice the loss is oscillating wildly instead of decreasing smoothly.

Name two possible causes and what you would try for each.

Answer: (1) Learning rate too high — the optimizer overshoots the minimum on each step. Fix: reduce the learning rate. (2) Batch size too small — the gradient estimate is very noisy, causing erratic updates. Fix: increase the batch size for smoother gradients. A third possibility: the loss landscape itself is very rugged, in which case switching to Adam (which adapts per-parameter learning rates) can help.

Q4: Backpropagation Efficiency

GPT-4 is estimated to have over one trillion parameters.

Without backpropagation, how many forward passes would you need to estimate the gradient for a single training step? Why is this infeasible?

Answer: You would need at least one forward pass per parameter to numerically estimate each partial derivative — over one trillion forward passes for a single update step. At any realistic speed, this would take years for one step. Backpropagation computes all gradients in a single backward pass, reducing the cost to roughly 2–3x a single forward pass regardless of parameter count.

Q5: Sharp vs. Flat Minima

Two networks achieve the same training loss. Network A sits in a sharp minimum; Network B sits in a flat minimum.

Which network would you expect to generalize better to new data, and why?

Answer: Network B (flat minimum) should generalize better. In a flat minimum, small perturbations to the weights — which naturally occur when the model encounters slightly different data — do not significantly change the loss. A sharp minimum is fragile: small weight changes can dramatically increase the loss. This is also why SGD noise and small batch sizes can help generalization — the noise pushes the optimizer away from sharp minima toward flatter regions.
