## Attention as a Learned Lookup

Attention is a differentiable database query. Every token asks a question, every token advertises what it offers, and the answer is a blend of the most relevant content.

:::columns cols="2" gap="40px"
**Traditional database**

Exact match on a key, returns one record. Hand-designed index, not differentiable.
+++
**Attention**

Soft match between query and key vectors, returns a weighted blend of all values. Fully differentiable, and the queries, keys, and values are all **learned**.
:::

## Introducing the Projections

$$Q = XW_Q, \qquad K = XW_K, \qquad V = XW_V$$

:::columns cols="2" gap="40px"
**Compare**

Dot products measure how well each query aligns with each key. A softmax over each row turns those scores into the **attention map** &mdash; one row of weights per token.
+++
**Retrieve**

Each token's output is its row of the attention map applied to the value vectors: a **weighted sum of V**.
:::
