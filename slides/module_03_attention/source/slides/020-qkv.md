**Traditional database**

Exact match on a key, returns one record. Hand-designed index, not differentiable.
+++
**Attention**

Soft match between query and key vectors, returns a weighted blend of all values. Fully differentiable, and the queries, keys, and values are all **learned**.
:::

## Shapes and Pairwise Compatibility

Consider 5 tokens with $d_{\text{model}} = 8$ and $d_k = 4$. The comparison step produces one score per **pair** of tokens:

$$\text{score}(i, j) = \mathbf q_i \cdot \mathbf k_j$$

:::columns cols="2" gap="30px"
**Tensor shapes**

| Tensor | Shape |
|---|---|
| $X$ | $(5, 8)$ |
| $W_Q, W_K, W_V$ | $(8, 4)$ |
| $Q, K, V$ | $(5, 4)$ |
| $QK^T$ (scores) | $(5, 5)$ |
+++
**Reading the score matrix**

The $(5, 5)$ matrix has one entry per pair. Row $i$ is token $i$'s query compared against every key.

A dot product measures alignment: when query $i$ points the same way as key $j$, the score is large. Training is what makes the model point queries toward the keys that matter.
:::
