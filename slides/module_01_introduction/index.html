The Chain Rule of Probability

How do we compute the probability of an entire sequence \(P(w_1, w_2, \ldots, w_n)\)?

$$P(w_1 w_2 \ldots w_n) = \prod_{k=1}^{n} P(w_k \mid w_1 \ldots w_{k-1})$$

This decomposes a joint probability into a product of conditional probabilities. But we still need to estimate \(P(w_k \mid w_1 \ldots w_{k-1})\), and we cannot count occurrences of every possible history.

The Markov Assumption

Instead of conditioning on the entire history, approximate with just the last few symbols:

$$P(w_k \mid w_1 \ldots w_{k-1}) \approx P(w_k \mid w_{k-n+1} \ldots w_{k-1})$$

For example, instead of computing:

\(P(\texttt{blue} \mid \texttt{The water of Walden Pond is so beautifully})\)

A bigram approximates this as:

\(P(\texttt{blue} \mid \texttt{beautifully})\)

This is the core trade-off of n-gram models: more context is better, but the data becomes sparser.

N-gram Model Estimation

Given the Markov assumption, we estimate each conditional probability by counting:

$$P(c_t \mid c_{t-n+1}, \ldots, c_{t-1}) = \frac{\text{count}(c_{t-n+1} \ldots c_t)}{\text{count}(c_{t-n+1} \ldots c_{t-1})}$$
  • Uniform random: \(P(c) = \frac{1}{27}\)
  • Unigram: \(P(c) = \frac{\text{count}(c)}{\text{total characters}}\)
  • Bigram: \(P(c_t \mid c_{t-1})\)
  • Trigram: \(P(c_t \mid c_{t-2}, c_{t-1})\)

Shannon described this exact procedure in 1948. Let's build it ourselves.

Shannon Entropy

How many bits do we need, on average, to encode a symbol?

$$H(X) = -\sum_{i} p(x_i) \log_2 p(x_i)$$

For a uniform distribution over 27 characters, every \(p(x_i) = \frac{1}{27}\):

$$H = -\sum_{i=1}^{27} \tfrac{1}{27} \log_2 \tfrac{1}{27} = -27 \cdot \tfrac{1}{27} \cdot \log_2 \tfrac{1}{27} = \log_2(27) = 4.75 \text{ bits/char}$$
  • This is the maximum entropy for 27 symbols. Any structure in the distribution lowers it.
  • English text: \(H \approx 1.0\text{--}1.5\) bits/char (Shannon, 1951). Most of each character is predictable from context.
  • The gap (4.75 vs ~1.2) = the redundancy that makes language models possible.

Cross-Entropy

How well does a model \(q\) predict text? For each character in the corpus, ask: how many bits does the model need to encode it?

$$H(q) = -\frac{1}{N} \sum_{i=1}^{N} \log_2 q(c_i \mid \text{context}_i)$$

This is the average number of bits per character the model needs. For each character \(c_i\), the model assigns a probability \(q(c_i \mid \text{context})\). Higher probability = fewer bits (\(-\log_2\) of a large number is small).

Perplexity converts this to an intuitive scale:

$$\text{Perplexity} = 2^{H(q)}$$

A perplexity of 10 means the model is as uncertain as choosing uniformly from 10 options. Lower = better.

Quiz

Test your understanding

Q1: Why Not Just Use Rules?

GOFAI systems used hand-written rules to process language. Why did this approach ultimately fail for general language understanding?

Answer: Language is inherently ambiguous and context-dependent. You cannot enumerate rules for every possible meaning of every word in every context. "Bank" alone has dozens of meanings, and the rules would need to cover all of them and all their interactions. Statistical/learned models discover these patterns from data instead of requiring manual specification.

Q2: Shannon's Entropy

Shannon estimated English has \(H \approx 1.0\text{--}1.5\) bits/char. A uniform 27-character alphabet has \(H = \log_2(27) = 4.75\) bits/char.

What does this gap tell us about English as a communication system?

Answer: English is highly redundant. About 70-80% of each character is predictable from context. This redundancy is what makes autocomplete, spell-check, and language models possible, and it is why you can read text with missing letters. Shannon showed this redundancy is a measurable, mathematical property.

Q3: The Memorization Problem

A word-level 5-gram model trained on Alice in Wonderland produces output that is mostly exact quotes from the book.

Why does this happen, and how do modern LLMs avoid it?

Answer: With high-order n-grams and limited training data, most contexts appear only once, so the model can only reproduce the original. Modern LLMs avoid this by (1) training on vastly more data, (2) using continuous representations instead of exact count tables, and (3) generalizing across similar contexts via learned embeddings.

Q4: Conditional Probability

A trigram model trained on English text has these counts for the context "th":

model["th"] = Counter({"e": 50, "a": 20, "i": 15, "o": 10, "u": 5})

What is \(P(\texttt{e} \mid \texttt{th})\), and what does it mean for the model's behavior?

Answer: \(P(\texttt{e} \mid \texttt{th}) = \frac{50}{50+20+15+10+5} = \frac{50}{100} = 0.5\). Half the time the model encounters the context "th", it will generate "e" as the next character. This is a conditional probability: it only applies when "th" is already the context. The other half of the time, the model generates "a" (20%), "i" (15%), "o" (10%), or "u" (5%). This is why trigram output produces "the" so frequently, but also generates "tha", "thi", and other combinations.

Q5: Compression and Prediction

Shannon proved that a perfect language model would also be a perfect compressor, and vice versa.

Why are prediction and compression fundamentally the same problem?

Answer: If you can predict the next symbol with high confidence, you need fewer bits to encode it: a symbol with probability \(p\) requires \(-\log_2(p)\) bits. A model that predicts perfectly (\(p = 1\)) needs 0 bits. Conversely, any compression scheme implicitly defines a probability model over the next symbol. Shannon's source coding theorem formalizes this: the minimum average code length equals the entropy \(H(X)\).
